# Project_2_Job_Search

## Описание
Проект Project_2_Job_Search - программа, которая получает
информацию о вакансиях с платформы hh.ru в России, сохранять
ее в файл и позволяет удобно работать с ней: добавлять, 
фильтровать, удалять. 

## Установка
- [клонируйте репозиторий](https://github.com/Grigorii-Goncharov/Project_2_Job_Search.git
)

- запустите файлы для обработки данных 
```
python main.py
```

## Использование
1. Перейдите на страницу в GitHub.
2. Создайте новую учетную запись или войдите в существующий.
3. Перенести проект в локальный репозиторий.
4. Откройте модул main.py и внесите свои данные для обработки данных.

## Структура проекта
```
PROJECT: Project_2_Job_Search
│
├──📁 data/
│   └──📄 vacancies.json          # JSON-файл с вакансиями
│
├──📁 src/                        # Исходный код
│   ├──📄 __init__.py             # Инициализация пакета
│   ├──📄 class_api.py            # API для получения вакансий
│   ├──📄 utils.py                # Вспомогательные функции
│   ├──📄 class_abstract.py       # Абстрактые Классы для работы с вакансиями
│   ├──📄 class_json.py           # Класс для работаы с JSON
│   └──📄 class_vacancy.py        # Класс для работы с вакансиями
│
├──📁 tests/                      # Тесты
│   ├──📄 __init__.py             # Инициализация тестов
│   ├──📄 conftest.py             # Фикстуры pytest
│   ├──📄 test_class_api.py       # Тесты классаAPI
│   ├──📄 test_class_json.py      # Тесты классаJSON
│   └──📄 test_class_vacancy.py   # Тесты класса Vacancy
│
├──📁 coverage/                   # Отчеты о покрытии
├──📄 .flake8                     # Конфиг линтера
├──📄 .gitignore                  # Игнорируемые файлы Git
├──📄 main.py                     # Точка входа
├──📄 poetry.lock                 # Лок файл зависимостей
├──📄 pyproject.toml              # Конфиг Poetry
└──📄 README.md                   # Документация
```

## Зависимости
Зависимости проекта управляются через Poetry. Они перечислены в файле `pyproject.toml`

## Тестирование
Для запуска тестов используйте команду: `paytest`

Для проверки покрытия кода тестами используйте команду: `pytest --cov`
